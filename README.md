# Cab-Hailing-Application
This is a very basic project, that deploys the Dijkstra's Algorithm to allocate cabs to customers and is built using C. This project was undertaken prior to acquiring proficiency in data structures, with the primary objective being, to attain a strong command of the C programming language.

Functional and Non - Functional Requirements:

Fixed Cab Database: Contains the information about the Drivers, Vehicles, Speeds, Default Location and the Ratings.
Fixed Map Database: Contains the information about the various location of the map and the distances between the
various locations. It must also contain the information about the surge times of each of the Routes that is being taken by the drivers.
User Input Interface: Switches the case between the various services offered like Advanced Booking, Cancellation of Trips and Normal Booking.
Booking: Books the cabs for the users according
to the current availability of the drivers filtered out by the Cab Finder Module.
Advanced Booking: Reserves the cabs for the
users based on their availability in the requested time period.
Cancellation: Cancels the trip for the user with
reference to the ID Assigned to the Trip.
Dynamic Location Tracker: Keeps track of the
current location of the drivers and the status of availability.
Advanced Availability Checker: Calculates the
availability of a driver based on his current trips and the requested time periods.
Trips Manager: Contains the information about
the ongoing trips of the database.
Availability Updater: Updates the data of available cabs after a booking or cancellation is done by a user.
Cab Finder Wizard: Filters out the best available options for the user according to the preferences given by the user.
Bill Generator: Generates the bill for the user based on the trips that are being booked or being canceled.
